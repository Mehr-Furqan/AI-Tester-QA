## PROFESSIONAL QA TESTING GUIDELINES

Based on ISTQB & industry best practices for comprehensive software testing.

### TESTING PYRAMID (Priority Order)

1. **Critical Path Testing** (70% of effort)
   - User authentication (login, logout, password reset)
   - Core business workflows
   - Payment/transaction processing
   - Data persistence
   - Error handling

2. **Feature Testing** (20% of effort)
   - All interactive elements
   - Form validation
   - Navigation flows
   - Modal/Dialog behavior
   - Notifications

3. **Non-Functional Testing** (10% of effort)
   - Performance
   - Accessibility
   - Cross-browser compatibility
   - Security basics

---

### MODAL & DIALOG TESTING CHECKLIST

**Appearance & Structure:**
- [ ] Modal displays centered on screen
- [ ] Modal has proper backdrop/overlay
- [ ] Content is readable and not clipped
- [ ] Modal doesn't exceed viewport height
- [ ] Scrollable if content > viewport
- [ ] Header/title clearly identifies purpose

**Functionality:**
- [ ] Close button is visible and clickable
- [ ] Escape key closes modal
- [ ] Clicking outside modal doesn't dismiss (or does if required)
- [ ] Form fields are functional inside modal
- [ ] Submit button works correctly
- [ ] Cancel/Close button cancels without saving

**User Interaction:**
- [ ] Focus management: focus moves to modal on open
- [ ] Tab order is logical inside modal
- [ ] Return focus to trigger element on close
- [ ] Keyboard navigation works (Tab, Shift+Tab, Enter, Escape)
- [ ] Mouse click areas are large enough

**Accessibility:**
- [ ] Modal has aria-modal="true"
- [ ] Modal has role="dialog" or role="alertdialog"
- [ ] Close button has aria-label="Close"
- [ ] Form fields have labels
- [ ] Error messages are announced to screen readers

**State Management:**
- [ ] Data entered in form is retained if closed without submit
- [ ] Reopening modal shows previous state or clean state (as expected)
- [ ] Form validation runs before submission
- [ ] Success message appears after submit
- [ ] Modal closes after successful action

**Edge Cases:**
- [ ] Opening multiple modals (nested handling)
- [ ] Modal on mobile/small viewport
- [ ] Modal with very long content
- [ ] Modal with form validation errors
- [ ] Modal with images/media that fail to load
- [ ] Modal with external links

---

### FORM TESTING CHECKLIST

**Field Validation:**
- [ ] Required fields show error when empty
- [ ] Email validation catches invalid formats
- [ ] Phone number format validation works
- [ ] Password strength requirements displayed
- [ ] Min/max length enforced
- [ ] Only valid characters accepted
- [ ] Special characters handled correctly

**User Input:**
- [ ] Placeholder text visible and helpful
- [ ] Tab between fields works
- [ ] Can paste content into fields
- [ ] Autocomplete works for common fields
- [ ] Backspace/Delete work as expected
- [ ] Copy-paste data validates correctly

**Form Submission:**
- [ ] Submit button disabled until valid
- [ ] Loading indicator shows during submission
- [ ] Success message displays
- [ ] Redirect to correct page after submit
- [ ] Error messages are specific and helpful
- [ ] Form can be resubmitted after error

**Error Handling:**
- [ ] Validation errors prevent submission
- [ ] Server errors show user-friendly messages
- [ ] No error exposures (technical stack traces)
- [ ] Error messages suggest fixes
- [ ] Failed field is highlighted

---

### NAVIGATION TESTING CHECKLIST

**Links & Buttons:**
- [ ] All links are clickable
- [ ] Links navigate to correct URL
- [ ] Button actions work as labeled
- [ ] Back button works correctly
- [ ] Forward button works after back
- [ ] Breadcrumbs navigate correctly

**Menu & Navigation:**
- [ ] All menu items visible/accessible
- [ ] Submenu items expand/collapse correctly
- [ ] Highlight shows current page
- [ ] Navigation doesn't break page layout
- [ ] Mobile menu opens/closes properly
- [ ] Menu closes when item clicked

**Page Flow:**
- [ ] Can reach all pages from home
- [ ] No broken links (404s)
- [ ] Page URLs are logical
- [ ] Browser history works
- [ ] Duplicate navigation doesn't create loops

### BUG SEVERITY & PRIORITY

**CRITICAL (P0):**
- App crash or complete failure
- Data loss or corruption
- Security vulnerability
- Authentication broken
- Payment processing failure
- Core feature completely unusable

**HIGH (P1):**
- Major feature not working
- Significant data issue
- Confusing/broken user flow
- Multiple users affected
- No workaround available

**MEDIUM (P2):**
- Feature partially broken
- Workaround exists
- Wrong calculation/result
- Unexpected but non-blocking error
- Minor UX confusion

**LOW (P3):**
- Cosmetic/visual issue
- Typo or grammar
- Alignment problem
- Minor performance lag
- Non-critical missing feature

### EFFECTIVE BUG REPORTING TEMPLATE

**Title:** [Component] Short description of issue
**Severity:** Critical | High | Medium | Low
**Steps to Reproduce:**
1. Navigate to [URL]
2. Click [element]
3. Enter [data]
4. [Expected result does not occur]

**Expected Result:** [What should happen]
**Actual Result:** [What actually happened]
**Screenshots/Videos:** [Attach evidence]
**Environment:** 
- Browser & Version
- OS
- Device (Desktop/Mobile)
- Timestamp

**Additional Context:**
- Does issue occur consistently? Always | Sometimes | Rarely
- Can you reproduce with different data?
- Does issue occur in other browsers?

### TESTING BEST PRACTICES

1. **Test with Real Data**
   - Use realistic names, emails, phone numbers
   - Test with international characters
   - Test with very long strings
   - Test with special characters

2. **Test Error Scenarios**
   - Invalid input immediately
   - Slow network
   - Missing network connection
   - Expired sessions
   - Insufficient permissions

3. **Test Boundaries**
   - Empty fields (null)
   - Very long input (500+ chars)
   - Minimum/maximum values
   - Special formatting (dates, numbers)

4. **Test User Journeys**
   - Happy path (everything works)
   - Sad path (errors occur)
   - Alternative paths (user makes different choices)

5. **Accessibility Testing**
   - Keyboard-only navigation
   - Screen reader compatibility
   - Color contrast
   - Focus indicators
   - ARIA labels

6. **Performance Testing**
   - Page load time < 3 seconds
   - Forms submit < 1 second
   - Animations smooth (60fps)
   - No memory leaks
   - Mobile performance acceptable

### QA MINDSET PRINCIPLES

✓ **Curiosity** - Always ask "what if?" and "why?"
✓ **Skepticism** - Don't trust it works until you verify
✓ **Attention to Detail** - Small issues compound
✓ **User Empathy** - Test like a real user would
✓ **Communication** - Report bugs clearly
✓ **Continuous Learning** - Platforms evolve, stay updated
✓ **Risk-Based** - Focus on high-impact areas first
✓ **Reproducibility** - Always provide clear repro steps

---

### COMMON QA PITFALLS TO AVOID

✗ Only testing happy path
✗ Assuming browser compatibility
✗ Skipping accessibility testing
✗ Not testing on mobile
✗ Reporting vague bugs
✗ Testing without understanding requirements
✗ Ignoring performance
✗ Not testing error scenarios
✗ Manual testing without documentation
✗ Forgetting to test logout/cleanup
